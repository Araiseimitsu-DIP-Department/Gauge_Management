# 変更履歴

## 2026-06-24

- 確認画面で確認済みバッチ選択時に `C-9` などのケースNoが欠落せず確認対象へ反映されるよう修正。
- 確認済みバッチ一覧の列名を「ケースNo」に変更し、確認対象一覧のID列を非表示化。
- 返却画面を上下積みレイアウトに変更し、返却日・返却ケースNo・選択サイズ表示・返却操作を返却対象一覧側へ集約。
- 返却画面の最大化表示では画面全体スクロールを抑制し、低解像度時のみ画面全体スクロールを許可するよう調整。
- 返却画面の機番入力で確定後に次項目・検索ボタンへフォーカス移動するよう変更し、返却対象一覧のID列を非表示化。
- 返却画面の検索ボタンのフォーカス表示を強調し、選択中サイズと個別返却・一括返却ボタンを返却入力行へ集約。
- 返却ケースNo未入力時は個別返却・一括返却を無効化し、入力が必要な旨を返却対象一覧に表示。
- 返却ケースNo欄の幅と未入力表示を調整し、選択中サイズ欄を右側の元の表示位置へ復帰。
- 返却ケースNo入力中とEnterキー確定時に、未入力表示と返却ボタンの有効状態が即時更新されるよう修正。
- 個別返却後は返却ケースNoを保持し、クリアボタンと一括返却後のみ返却ケースNoをクリアするよう変更。
- 貸出登録のサイズ入力を1〜10、11〜20、21〜30の縦3列表示へ変更し、自動フォーカス移動は番号順を維持。
- 貸出登録で機番左、機番右、担当者、サイズ1の順に自動フォーカス移動するよう変更。
- 確認画面で選択中サイズ表示と個別確認ボタンを追加し、確認済みバッチの列名を「返却ケースNo」に変更。
- 確認対象検索のラベルを「返却ケースNo」に変更し、入力欄・検索ボタン・確認操作行の配置をグリッドで整列。
- 確認済みバッチの削除ボタンを画面から削除。
- 個別確認後は返却ケースNoと確認対象一覧を保持し、直前の表示内容が残るよう変更。
- Access から PostgreSQL へ再取込を実施し、README に画面運用メモと移行結果を追記。
- ターミナルから起動したアプリを中断した際に `KeyboardInterrupt` の traceback を表示せず終了するよう調整。
